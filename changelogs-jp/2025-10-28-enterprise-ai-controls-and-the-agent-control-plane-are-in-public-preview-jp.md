# エンタープライズ AI コントロールとエージェントコントロールプレーンがパブリックプレビューで利用可能

## Overview
AI システムの管理機能に焦点を当てた新しいエンタープライズレベルのエクスペリエンスを発表します。これは、Copilot を使用するすべての GitHub Enterprise Cloud のお客様が本日から利用できます。これはエージェントコントロールプレーンであり、組織が環境全体でエージェントがどのように動作するかをより深く制御できるように設計されたエンタープライズガバナンス機能のスイートです。

## Detailed Explanation
### 概要
- AI システムの管理機能に焦点を当てた新しいエンタープライズレベルのエクスペリエンスを発表します。これは、Copilot を使用するすべての GitHub Enterprise Cloud のお客様が本日から利用できます。これはエージェントコントロールプレーンであり、組織が環境全体でエージェントがどのように動作するかをより深く制御できるように設計されたエンタープライズガバナンス機能のスイートです。
- エンタープライズ管理者は、ビジネスを安全に保ちながらチームの AI 機能を有効にする責任を負っています。この仕事を上手く遂行するために、エージェントアクティビティの認識を改善し、監査可能性を高め、カスタムエージェントをエンタープライズ対応にする補完機能を起動しています。エンタープライズ内でエージェントが何をしているかについての質問に答えることがこれまでになく簡単になりました。

### AI コントロールはエージェントコントロールプレーンにあります
- エージェントコントロールプレーンの基本機能から始めましょう：
- GitHub の AI システムに関連するすべての管理タスクのための統合ビューとトップレベルナビゲーション。
- 管理者はエージェントフリートを管理できます。現在の少数のエージェント、そして将来の数千のエージェントが、エンタープライズのためにここで運用されます。
- Copilot および AI 関連システムのエンタープライズレベルのポリシーと設定の恒久的なホーム。

### エンタープライズ対応のカスタムエージェント
- カスタムエージェントの起動により、エンタープライズは標準化された採用への道を決定したいと考えるでしょう：
- カスタムエージェントのエンタープライズ標準をホストする1つの組織を設定します。
- 1クリックのプッシュルールを使用して、カスタムエージェントの静的ファイルパス（つまり、.github/agents/*.md）をエンタープライズ全体で編集から保護します。
- 組織の設計と実験の許容範囲に合わせてプッシュルールを調整します。
- 任意の Issue に Copilot coding agent を割り当てる際に、エンタープライズ定義のカスタムエージェントを選択します。

### エージェントセッションアクティビティ
- エンタープライズ内のエージェントアクティビティの認識を高めます：
- 過去24時間のエンタープライズ全体のエージェントセッションを確認します。
- エージェントタイプとタスク状態（例：完了、キャンセル、進行中）でフィルタリングします。
- 権限がある場合は、ターゲットリポジトリに直接移動します。

### エージェント監査ログ
- エージェントは実際に何をしているのでしょうか？監査ログに答えがあります：
- エージェントアクティビティの新しいフィールド（例：Copilot によって実行された pull_request.create）：actor_is_agent を識別します。エージェントが誰の代わりに行動しているかを識別するために user と user_id を表示します。
- agent_session.task の新しいイベントは、どのセッションが開始、終了、または完了に失敗したかを示します。
- AI controls タブからエージェント関連の監査ログに素早く移動できます。
- actor_is_agent を識別します。
- エージェントが誰の代わりに行動しているかを識別するために user と user_id を表示します。

### AI 管理のきめ細かい権限（FGP）が近日公開！
- AI と Copilot の管理は、ほとんどの企業で人々のチームによって行われる仕事です。そして、そのチームが常に GitHub Enterprise 全体を実行する責任を負っているわけではありません。今週後半には次のことができます：
- AI controls ビューへのアクセスを許可する新しい FGP を利用します。
- 広範なエンタープライズ所有権を付与することなく AI システムの管理を分散化するエンタープライズカスタムロールを構築します。
- エンタープライズカスタムロールをエンタープライズチームに割り当てて、規模で管理します。

### MCP エンタープライズ許可リスト
- VS Code Insiders での MCP 接続を管理するために、MCP レジストリ URL を介してエンタープライズ全体の MCP 許可リストを設定します。
- AI controls エクスペリエンスから MCP ポリシーを設定します。
- 信頼できないツールの使用を防ぐために、レジストリからローカルサーバーを省略します。
- VS Code および他の IDE の一般提供のサポートは近日公開されます！

- GitHub Community でディスカッションに参加してください。
