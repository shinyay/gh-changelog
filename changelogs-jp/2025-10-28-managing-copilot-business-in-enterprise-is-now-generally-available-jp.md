# エンタープライズでの Copilot Business の管理が一般提供開始

## Overview
GitHub Enterprise Cloud のエンタープライズオーナーは、組織へのアクセス権を付与することなく、エンタープライズアカウント内で Copilot Business ライセンスを割り当てることができるようになりました。このエクスペリエンスは、ユーザーごとに GitHub Enterprise Cloud ライセンスを必要とせずに、より柔軟でスケーラブルな Copilot Business ライセンスモデルをエンタープライズに提供します。

## Detailed Explanation
### Overview
- GitHub Enterprise Cloud のエンタープライズオーナーは、組織へのアクセス権を付与することなく、エンタープライズアカウント内で Copilot Business ライセンスを割り当てることができるようになりました。このエクスペリエンスは、ユーザーごとに GitHub Enterprise Cloud ライセンスを必要とせずに、より柔軟でスケーラブルな Copilot Business ライセンスモデルをエンタープライズに提供します。
- エンタープライズアカウントを通じて Copilot Business ライセンスを割り当てるには、既存の組織メンバーか非関連ユーザーのいずれかとして、ユーザーがエンタープライズにすでに存在している必要があります。ユーザーに直接割り当てる場合でも、エンタープライズチームを通じて割り当てる場合でも、エンタープライズオーナーは専用の Copilot Business ライセンスページを使用してこれらのライセンスを割り当てることができます。ユーザーが組織を通じてもエンタープライズを通じて直接も割り当てられている場合、1 つのライセンスのみを消費し、割り当てられた最上位の Copilot ライセンスにアクセスできます。
- 新しい専用ライセンスページは、エンタープライズオーナーに次の機能を提供します:
- エンタープライズの現在の Copilot ライセンス消費の概要。
- エンタープライズとその組織全体で Copilot を有効にする一元化された場所。
- 個々のユーザーに対する Copilot Business の直接割り当てを管理する機能
- エンタープライズチームに Copilot Business を割り当てる機能: エンタープライズチームは現在パブリックプレビューであり、新しいライセンスエクスペリエンスと並行して使用できます。
- GitHub トライアルから開始する新しいエンタープライズの場合、エクスペリエンスが更新され、エンタープライズオーナーが初日からこの新しいライセンスエクスペリエンスを試せるようになりました。エンタープライズオーナーは、エンタープライズライセンスで割り当てられた Copilot Business ライセンスのみでトライアルを変換し、必要に応じて後で GitHub Enterprise Cloud ライセンスを追加できます。
- 詳細については、Copilot Business ライセンス、エンタープライズチーム、非関連ユーザーに関するドキュメントをご覧ください。この新しいエクスペリエンスについてフィードバックがある場合は、このディスカッションにご参加ください。

## Key Changes
- エンタープライズアカウント内で組織アクセスなしに Copilot Business ライセンスを割り当て可能
- 専用の Copilot Business ライセンスページで一元管理
- エンタープライズチームへのライセンス割り当てをサポート(パブリックプレビュー)
- トライアルからの変換時に Copilot Business ライセンスのみで開始可能

## Impact / Who's Affected
- GitHub Enterprise Cloud のエンタープライズオーナーが対象
- ユーザーが組織とエンタープライズの両方を通じて割り当てられている場合、1 つのライセンスのみを消費
- エンタープライズチームは現在パブリックプレビュー

## Caveats / Limitations
- ユーザーがエンタープライズにすでに存在している必要があります(組織メンバーまたは非関連ユーザーとして)
- エンタープライズチーム機能は現在パブリックプレビュー
