# GitHub AI ツールの予算追跡で AI 支出をコントロール

## Overview
AI を活用した開発ワークフローが拡大するにつれて、それらを効果的に管理する能力も拡大すべきです。本日、GitHub Copilot コーディングエージェントと GitHub Spark 用の2つの新しいプレミアムリクエスト SKU、およびコスト管理をシンプルかつ予測可能にする新しいバンドル予算オプションを開始します。

## Detailed Explanation
### Overview
- AI を活用した開発ワークフローが拡大するにつれて、それらを効果的に管理する能力も拡大すべきです。本日、GitHub Copilot コーディングエージェントと GitHub Spark 用の2つの新しいプレミアムリクエスト SKU、およびコスト管理をシンプルかつ予測可能にする新しいバンドル予算オプションを開始します。

### Granular visibility meets flexible control
- 明確な帰属のための専用 SKU GitHub Copilot コーディングエージェントと GitHub Spark に専用のプレミアムリクエスト SKU が追加されました。既存の Copilot Premium Request SKU は、プレミアムリクエストを消費する他のすべてのツールの使用状況を引き続き追跡します。
- プレミアムリクエスト予算戦略を選択 詳細な制御を好むか、合理化された管理を好むかにかかわらず、対応しています：
- 製品固有の予算：SKU レベルまで個別の予算を設定して、各 AI ツールに異なる支出制限を定義します。
- バンドル予算：すべてのプレミアムリクエスト SKU の支出を1つの統一された予算内で管理します。これは、見落としなくシンプルさを求めるチームに最適です。
- 予算の設定について詳しく学んでください。
- 強化されたポリシーコントロール（Enterprise および Team プランのみ）新しいプレミアムリクエスト超過ポリシーオプションにより、エンタープライズおよびチーム管理者はツールごとに超過を有効または無効にできます。コーディングエージェントの超過を許可したいが Spark では許可したくない場合は、Copilot ポリシー内で数秒で構成できます。
- プレミアムリクエストポリシーの構成について詳しく学んでください。
- すでに構成した予算への影響なし 既存のプレミアムリクエスト予算は、新しいバンドル予算システムに自動的に移行されます。これにより、すでに構成されたコスト保護がそのまま維持されます。
- Enterprise および Team プラン：$0 のプレミアムリクエスト予算がある場合、11月18日に削除されます。詳細については、この削除に関する changelog をご覧ください。

### Why this matters
- 予算を意識するチームのために：どの AI ツールが使用を促進しているかについての推測をやめてください。
- スケーリング組織のために：必要な詳細な支出管理で、自信を持って AI 採用を拡大してください。
- 成長の基盤を構築するために：GitHub の AI エコシステムが成長し続けるにつれて、高度な AI ガバナンスを実現します。

### What you need to know
- 履歴データは変更なし：このリリース前の Spark とコーディングエージェントの使用状況は、Copilot Premium Requests SKU のままです。今後のすべての新しい使用状況は、正しい SKU に適切に帰属されます。
- Pro/Pro+ の $0 予算は継続：Pro および Pro+ ユーザーは、$0 予算で引き続き支出を管理します。プレミアムリクエスト超過ポリシーは、現在 GitHub Copilot Business および GitHub Copilot Enterprise アカウントでのみ利用可能です。
- バンドル予算は将来に対応：プレミアムリクエストを使用する新しいツールを導入すると、それらは自動的にバンドル予算に追加されるため、予期しない料金について心配する必要はありません。
- GitHub Community で、これらの新機能についてフィードバックを共有したり、質問したりしてください。

## Impact / Who's Affected
