# npm セキュリティアップデート：クラシックトークン作成の無効化と詳細トークンの変更

## Overview
編集者注（2025年11月5日）：影響を受けるトークンが npm トークンであることを明示的に明確化するため、この投稿を更新しました。

## Detailed Explanation
### 概要
- 編集者注（2025年11月5日）：影響を受けるトークンが npm トークンであることを明示的に明確化するため、この投稿を更新しました。
- 本日、npm のセキュリティを強化する継続的な取り組みにおける、もう一つのマイルストーンを迎えました。以前発表したとおり、npm のトークン管理システムの最初の変更セットを実装しています。
- 重要：これらの変更は、npm レジストリに使用される npm トークンにのみ影響します。GitHub トークン（例：個人アクセストークン、詳細トークンなど）は、これらの変更による影響を受けません。

### npm クラシックトークン
- 新しい npm クラシックトークンは、npmjs.com ウェブサイト、CLI、または API を介して作成できなくなりました。
- 既存の npm クラシックトークンは、2025年11月19日まで引き続き機能します。
- npm token create CLI コマンドは、npm クラシックトークンを生成しなくなりました。

### npm 詳細アクセストークン
- 書き込み権限を持つ新しい npm トークンは、デフォルトで 2FA を適用するようになりました。
- CI/CD ワークフロー用の新しい「Bypass 2FA」オプションが利用可能になりました（デフォルトではチェックされていません）。
- 書き込み権限を持つすべての npm 既存詳細トークンは、90日間の最大有効期間が設定されるようになりました。2026年2月3日以降に期限切れになるように設定されている既存トークンは、その日に期限切れになるよう調整されました。
- 2026年2月3日以降に期限切れになるように設定されている既存トークンは、その日に期限切れになるよう調整されました。

### npm クラシックトークンを使用している場合
- 2025年11月19日までに npm 詳細アクセストークンに移行する必要があります。この日以降、すべての npm クラシックトークンは永久に取り消されます。
- npmjs.com/settings/~/tokens にアクセスします。
- 「Generate New Token」→「Granular Access Token」をクリックします。
- ユースケースに適した権限を設定します。
- 書き込みアクセスが必要な CI/CD ワークフローの場合、「Bypass 2FA」を有効にするか、OIDC を使用した確立されたトラストパブリッシングを使用することを検討してください。
- ワークフロー内の npm クラシックトークンを新しい npm 詳細トークンに置き換えます。
- 注：npm 詳細トークンは現在、npmjs.com ウェブサイトを介して作成する必要があります。今後数週間で、npm token コマンドを介した詳細トークン管理の完全な CLI サポートを追加するよう積極的に取り組んでいます。

### npm 詳細トークンを使用している場合
- npmjs.com/settings/~/tokens で npm トークンの有効期限を確認します。
- 2026年2月3日に期限切れになるトークンがある場合、トークンローテーションを計画します。
- CI/CD 用に作成された新しいトークンで、非対話型のパブリッシングが必要な場合、「Bypass 2FA」の有効化が必要になる場合があります。

### これらの変更による影響を受けないもの
- GitHub 個人アクセストークン（クラシック）
- GitHub 詳細個人アクセストークン
- GitHub Actions シークレット — 影響はありませんが、シークレットとして保存されている npm トークンは更新する必要があります
- アクションの GITHUB_TOKEN

### 今後の予定：2025年11月19日
- 11月19日に、すべての npm クラシックトークンを永久に取り消し、長期間有効なローカルパブリッシングトークンを2時間のセッショントークンに置き換えます。
