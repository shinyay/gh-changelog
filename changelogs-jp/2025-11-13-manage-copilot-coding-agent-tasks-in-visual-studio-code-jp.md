# Visual Studio CodeでCopilotコーディングエージェントのタスクを管理

## Overview
Visual Studio Codeを離れることなく、Copilotコーディングエージェントを追跡・管理できるようになりました。必要なものすべてが、一元化されたミッションコントロールビューに集約されています。

## Detailed Explanation
### Overview
- Visual Studio Codeを離れることなく、Copilotコーディングエージェントを追跡・管理できるようになりました。必要なものすべてが、一元化されたミッションコントロールビューに集約されています。
- エージェントセッションパネルをクリックすることで、VS Code内でセッションログを直接表示できます。Copilotのコミットの背後にある理由を、コンテキスト内でリアルタイムに確認できます。

### Centralizing your task management
- 新しいタスクビューにより、タスク間をシームレスに切り替えられます。タスクのステータスを一目で確認し、Copilotが入力を必要とするときにすぐに対応できます。

### Continue where you prefer working
- Copilotの作業を基に構築したい場合でも、完全にコントロールしたい場合でも、VS Code、Codespaces、またはGitHub CLIでセッションを開けます。github.comのセッション表示ページで「Open in VS Code」をクリックすると、VS Codeワークスペースで直接セッションを開けます。
- この機能は、Copilotコーディングエージェントへのアクセス権を持つすべてのユーザーが利用できます。Copilotとタスクでコラボレーションするには、リポジトリへの書き込みアクセスが必要です。VS Codeでセッションにアクセスするには、最新バージョンのVS CodeとGitHub Copilot Chat、GitHub Pull Requests拡張機能がインストールされていることを確認してください。
- GitHub Communityのディスカッションに参加して、フィードバックを共有してください。

## [Continue where you prefer working](#continue-where-you-prefer-working)


Copilotの作業を基に構築したい場合でも、完全にコントロールしたい場合でも、VS Code、Codespaces、またはGitHub CLIでセッションを開けます。github.comのセッション表示ページで**Open in VS Code**をクリックすると、VS Codeワークスペースで直接セッションを開けます。


![Screenshot showing Open in VS Code on github.com](https://github.com/user-attachments/assets/6a54dae1-9d1b-468c-ac71-5fc16d7bdb70)


この機能は、Copilotコーディングエージェントへのアクセス権を持つすべてのユーザーが利用できます。Copilotとタスクでコラボレーションするには、リポジトリへの書き込みアクセスが必要です。VS Codeでセッションにアクセスするには、最新バージョンのVS CodeとGitHub Copilot Chat、[GitHub Pull Requests](https://marketplace.visualstudio.com/items?itemName=GitHub.vscode-pull-request-github)拡張機能がインストールされていることを確認してください。
