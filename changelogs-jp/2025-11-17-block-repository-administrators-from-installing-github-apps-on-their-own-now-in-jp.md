# リポジトリ管理者によるGitHub Appsの独自インストールをブロックする機能がパブリックプレビューに

## Overview
組織オーナーは、パブリックプレビューで利用可能な新しい設定により、どのユーザーがリポジトリ全体にGitHub Appsをインストールできるかをより適切に制御できるようになりました。

## Detailed Explanation
### Overview
- 組織オーナーは、パブリックプレビューで利用可能な新しい設定により、どのユーザーがリポジトリ全体にGitHub Appsをインストールできるかをより適切に制御できるようになりました。
- 以前は、外部コラボレーターを含むリポジトリの管理者権限を持つすべてのユーザーが、リポジトリレベルの権限のみを要求するGitHub Appsをインストールできました。これにより、予期しないアプリのインストールやガバナンスの課題が発生することがあり、特に厳格なセキュリティまたはコンプライアンス要件を持つ組織では問題となっていました。
- この新しい設定により、組織オーナーはリポジトリ管理者が管理するリポジトリにGitHub Appsをインストールできないようにすることができます。設定が有効になっている場合、組織リポジトリにアプリをインストールできるのは組織オーナーのみになります。リポジトリ管理者は、代わりに組織オーナーにアプリのインストールを要求する必要があります。
- このメリット
- 信頼できるユーザーのみがリポジトリ全体にGitHub Appsをインストールできるようにすることで、組織のガバナンスを強化します。
- 未承認または予期しないアプリのインストールのリスクを軽減します。
- コンプライアンス要件の遵守を支援します。
- 開始するには、組織オーナーは組織の設定に移動し、「Member privileges」タブ内の「GitHub Apps」で新しいオプションを探してください。
