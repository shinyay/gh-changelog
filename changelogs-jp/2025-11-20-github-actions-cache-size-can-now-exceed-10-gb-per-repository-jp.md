# GitHub Actionsのキャッシュサイズがリポジトリあたり10 GBを超えることが可能になりました

## Overview
拡張されたGitHub Actionsキャッシュストレージにより、ワークフロー実行間でより多くのビルド依存関係を保存できるようになりました。リポジトリは、従量課金モデルを使用して以前のリポジトリあたり10 GBのキャッシュ上限を超えることができます。すべてのリポジトリは引き続き10 GBにアクセスでき、これは追加費用なしで提供されます。エンタープライズ、組織、リポジトリの管理者はこれらの制限を増やすことができます。これらの制限を増やすと、キャッシュはリポジトリあたり10 GBを超えることができ、使用されたストレージに基づいて自動的に課金されます。このストレージコストは、Git LFSやCodespacesストレージのストレージと同等であり、リポジトリあたり10 GBの制限を超えるには、有効なPro、Team、またはEnterpriseアカウントが必要です。

## Detailed Explanation
### GitHub Actions cache size can now exceed 10 GB per repository
- 拡張されたGitHub Actionsキャッシュストレージにより、ワークフロー実行間でより多くのビルド依存関係を保存できるようになりました。リポジトリは、従量課金モデルを使用して以前のリポジトリあたり10 GBのキャッシュ上限を超えることができます。すべてのリポジトリは引き続き10 GBにアクセスでき、これは追加費用なしで提供されます。エンタープライズ、組織、リポジトリの管理者はこれらの制限を増やすことができます。これらの制限を増やすと、キャッシュはリポジトリあたり10 GBを超えることができ、使用されたストレージに基づいて自動的に課金されます。このストレージコストは、Git LFSやCodespacesストレージのストレージと同等であり、リポジトリあたり10 GBの制限を超えるには、有効なPro、Team、またはEnterpriseアカウントが必要です。

### Managing cache and billing policies
- 2つのキャッシュ管理ポリシーを導入しています：キャッシュサイズ削除制限（GB）とキャッシュ保持制限（日数）。保持制限を使用して、最終アクセス後にキャッシュエントリが保持される期間を設定し、サイズ削除制限を使用して各リポジトリの最大合計キャッシュサイズを設定します。デフォルトでは、追加費用なしで10 GBのキャッシュサイズ制限と7日間の保持制限を受け取ります。プランのデフォルトを超えてキャッシュサイズ制限を増やすと、追加のキャッシュストレージに対して課金されます。リポジトリのキャッシュが制限を超えると、最も最近使用されていないキャッシュエントリが自動的に削除され、設定されたサイズ内に収まります。デフォルトを維持する場合、何も変わりません。
- エンタープライズ、組織、リポジトリの管理者は、キャッシュポリシーを管理して使用されるキャッシュの量を制御できます。これらのポリシーは、Actionsの設定（リポジトリと組織の場合）またはポリシー（エンタープライズの場合）を介して管理でき、ポリシーはカスケードします（例：エンタープライズの最大値がリポジトリあたり15 GB、14日間の場合、そのエンタープライズ内のすべての組織の最大値はリポジトリあたり15 GB、14日間です）。
- さらに、請求所有者はこの新しいSKUに基づいて予算を設定できます。予算に達した場合、より高い制限を使用しているすべてのリポジトリで、次の請求サイクルまでキャッシュは読み取り専用になります。
- GitHub Actionsドキュメントでキャッシュストレージの管理について詳しく学びましょう。
