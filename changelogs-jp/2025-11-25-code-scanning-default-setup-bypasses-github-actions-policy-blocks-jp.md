# コードスキャニングのデフォルト設定がGitHub Actionsポリシーブロックをバイパスするようになりました

## Overview
GitHubコードスキャニングのデフォルト設定は、組織にどのワークフローを実行できるかを制限するGitHub Actionsポリシーがある場合でも実行されるようになりました。過去には、制限的なアクションポリシーがコードスキャニングのデフォルト設定の実行をブロックし、セキュリティカバレッジを低下させる可能性がありました。このアップデートにより、どのGitHub Actionsワークフローを実行するかを制限できますが（例：エンタープライズで承認されたワークフローのみを許可）、GitHub Actionsが有効である限り、コードスキャニングのデフォルト設定ワークフローは引き続き実行されます。

## Detailed Explanation
### Overview
- GitHubコードスキャニングのデフォルト設定は、組織にどのワークフローを実行できるかを制限するGitHub Actionsポリシーがある場合でも実行されるようになりました。過去には、制限的なアクションポリシーがコードスキャニングのデフォルト設定の実行をブロックし、セキュリティカバレッジを低下させる可能性がありました。このアップデートにより、どのGitHub Actionsワークフローを実行するかを制限できますが（例：エンタープライズで承認されたワークフローのみを許可）、GitHub Actionsが有効である限り、コードスキャニングのデフォルト設定ワークフローは引き続き実行されます。
- このワークフローに影響を与える唯一のアクションポリシーは、Disable actionsです。GitHub Actionsが組織またはリポジトリレベルで無効になっている場合、コードスキャニングのデフォルト設定は実行されません。
- この変更は現在、すべてのGitHubプランで一般提供されており、アクションは必要ありません。Enterprise Serverの3.19で提供される予定です。
- 詳細については、コードスキャニングのデフォルト設定のドキュメントを参照してください。
