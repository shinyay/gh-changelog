# リポジトリ管理者によるGitHub Appsのインストールをブロックする機能が一般提供開始

## Overview
組織所有者は、新しい設定により、リポジトリ全体でGitHub Appsをインストールできるユーザーをより適切に管理できるようになりました。

## Detailed Explanation
### Overview
- 組織所有者は、新しい設定により、リポジトリ全体でGitHub Appsをインストールできるユーザーをより適切に管理できるようになりました。
- 以前は、外部コラボレーターを含むリポジトリの管理者権限を持つすべてのユーザーが、リポジトリレベルの権限のみを要求するGitHub Appsをインストールできました。これにより、予期しないアプリのインストールやガバナンスの課題が発生する可能性があり、特に厳格なセキュリティやコンプライアンス要件を持つ組織にとって問題となっていました。
- この新しい設定により、組織所有者はリポジトリ管理者が管理するリポジトリへのGitHub Appsのインストールを防ぐことができます。この設定を有効にすると、組織所有者のみが組織のリポジトリにGitHub Appsをインストールできるようになります。リポジトリ管理者は、代わりに組織所有者にアプリのインストールをリクエストする必要があります。
- メリット
- 信頼できるユーザーのみがリポジトリ全体でGitHub Appsをインストールできるようにすることで、組織のガバナンスを強化できます。
- 不正または予期しないアプリのインストールのリスクを軽減します。
- コンプライアンス要件を満たすのに役立ちます。
- 開始するには、組織所有者は組織の設定に移動し、メンバー権限タブをクリックして、GitHub Appsの下に新しいオプションを見つけることができます。
