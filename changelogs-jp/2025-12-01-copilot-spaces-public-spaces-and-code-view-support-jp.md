# Copilot Spaces: パブリックスペースとコードビューのサポート

## Overview
Copilot Spacesに3つの主要なアップデートをリリースしました。パブリックスペース、個別共有、github.comコードビューアーから直接スペースにファイルを追加する機能です。

## Detailed Explanation
### Overview
- Copilot Spacesに3つの主要なアップデートをリリースしました。パブリックスペース、個別共有、github.comコードビューアーから直接スペースにファイルを追加する機能です。

### Public spaces
- リンクを持つ誰でもアクセスできるパブリックスペースを作成できるようになりました。これは以下の用途に最適です。
- オープンソースのドキュメントとリソース。
- GitHubコミュニティとテンプレートやワークフローを共有する。
- 組織間のコラボレーション。
- 教育用デモ。
- パブリックスペースは個人所有のスペースにのみ利用可能で、組織所有のスペースには利用できません。コンテンツの整合性を保護するため、デフォルトで閲覧専用です。アクセス制御を追加するため、個別共有機能を追加しました。

### Individual sharing
- 個人所有のスペースを共有できるようになりました。これにより、軽量な個人ワークフローと個人間の柔軟なコラボレーションが可能になります。

### Privacy & security
- このリリースは、GitHubのセキュリティと信頼の基準を維持しています。
- プライベートまたは機密コンテンツが公開されることはありません。パブリックスペースと個別共有は、閲覧者が既にアクセス許可を持っているコンテンツのみを表示します。
- ロールベースのアクセス制御は常に適用されます。パブリックスペースでも同様です。
- アクセスの変更は即座に反映されます。常に完全な制御が可能です。
- パブリックリンクにはサインインは不要ですが、閲覧者は既に認可されているものだけを見ることができます。
- パブリックスペースは直接リンクを介してのみ検出可能です。インデックス化されません。

### 📎 Add files to a space directly from the code viewer
- github.comコードビューアーから直接スペースにファイルを追加できるようになりました。この組み込みエントリポイントは、自然なワークフローに適合し、コンテキストの切り替えを削減します。

### Security & permissions
- ユーザーはアクセス許可のあるファイルのみを追加できます。
- スペースレベルのロールベースのアクセス制御とリポジトリのアクセス制御は引き続き適用されます。

### 💬 Get started & resources
- 今すぐgithub.com/copilot/spacesでCopilot Spacesを試してください。
- 一般提供開始のアナウンスでCopilot Spacesの機能について詳しく学ぶ
- UI内のインラインのフィードバックを送信ボタンを使用してフィードバックを提供してください。
- Copilot Spacesのドキュメント。
- Copilot Spacesの課金モデル。
