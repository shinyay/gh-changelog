# リポジトリダッシュボード：検索、保存クエリがプレビューで利用可能に

## Overview
適切なリポジトリを見つけることは、干し草の中から針を探すように感じることがあります。複数のプロジェクトを扱っている場合や、チーム間でコラボレーションしている場合、リポジトリの追跡は圧倒的になります。

## Detailed Explanation
### Overview
- 適切なリポジトリを見つけることは、干し草の中から針を探すように感じることがあります。複数のプロジェクトを扱っている場合や、チーム間でコラボレーションしている場合、リポジトリの追跡は圧倒的になります。
- 新しいリポジトリダッシュボードプレビューにより、アクセスできるリポジトリを簡単に見つけて、保存されたビューを作成できます。すべてが一箇所に集約されています。

### 仕組み
- プルリクエストとissuesダッシュボードと同様に、reposダッシュボードはリポジトリ発見のコマンドセンターとして機能します。検索機能、フィルタリング、保存されたクエリ、および貢献とアクセスに基づいてリポジトリを優先順位付けする関連性ベースのソートアルゴリズムを組み合わせています。
- reposダッシュボードには3つの方法でアクセスできます：
- github.com/reposに直接アクセス
- トップグローバルナビゲーションバーのリポジトリアイコンをクリック
- 左側のナビゲーションメニューからRepositoriesを選択

### 利点
- 一箇所でリポジトリを発見：個人プロジェクトからエンタープライズコードベースまで、貢献したりアクセスできるリポジトリを見つけます。
- 厳選されたビューで開始：デフォルトビューで最も重要なリポジトリに即座にアクセス：`My contributions`、`My repositories`、`My forks`、`Adminable by me`。
- フィルタリングで絞り込む：`visibility`、`language`、`organization`などのフィルターを使用して検索を絞り込みます。関連性でソートして、最もアクティブなリポジトリを表示します。
- カスタムクエリを保存：カスタムフィルターをブックマークして、ワークフローに最も重要なリポジトリビューに即座に戻ります。

### 制限事項
- 関連性フィルターと`My contributions`ビューには、過去1年間のアクティビティのみが含まれます。1年以上前に貢献したリポジトリはこれらのビューに表示されませんが、これらのリポジトリは検索や他のフィルターを使用して見つけることができます。
- 関連性には、リポジトリのissue、プルリクエスト、ディスカッションに対するコメントからの重み付けが含まれます。コード貢献のみに依存しません。この重み付けは継続的に改善している領域であるため、「適切」に感じない場所を共有してください。改善を続けることができます。
- リポジトリのカスタムプロパティでフィルタリングするには、プロパティの前に組織名を付けて、プロパティと値を手動で入力します。将来のアップデートでプロパティと値の自動提案を追加する予定です。サンプルクエリ：`org:Byte-Sized-Demo props.production:true`
- リポジトリダッシュボードにアクセスして、GitHubコミュニティのディスカッションに参加してフィードバックを共有してください。

## Impact / Who's Affected
- 制限事項 関連性フィルターと`My contributions`ビューには、過去1年間のアクティビティのみが含まれます。

## Caveats / Limitations
