# GitHub Copilot のエージェントメモリがパブリックプレビューに

## Overview
Copilot メモリが、すべての有料 GitHub Copilot プランでパブリックプレビューとして利用可能になりました。

## Detailed Explanation
### Overview
- Copilot メモリが、すべての有料 GitHub Copilot プランでパブリックプレビューとして利用可能になりました。
- Copilot メモリにより、Copilot はリポジトリに関する有用な詳細を学習して保持できるようになります。Copilot コーディングエージェント、Copilot コードレビュー、または Copilot CLI で作業すると、時間の経過とともにサポートの質が向上するリポジトリ固有の理解が構築されます。

### How it works
- Copilot は、リポジトリで作業する際に「メモリ」と呼ばれる厳密にスコープされた洞察を自動的にキャプチャします。これらのメモリは:
- リポジトリ固有であり、使用前に現在のコードベースと照合されます。
- Copilot の機能間で共有されるため、コーディングエージェントが学習した内容がコードレビューに役立ちます。
- 古い情報を防ぐために 28 日後に自動的に期限切れになります。
- Copilot メモリの正確な動作の詳細については、Copilot メモリに関するエンジニアリングブログ記事をご覧ください。

### Get started
- 個人ユーザー (Copilot Pro または Pro+): 個人の Copilot 設定で Copilot メモリを有効にします。
- 組織およびエンタープライズ : 組織またはエンタープライズ設定のポリシー設定を通じて有効にします。
- リポジトリオーナーは、Repository Settings > Copilot > Memory で保存されたメモリを確認および削除できます。
- この機能の使用方法の詳細については、Copilot メモリのドキュメントをご覧ください。
- GitHub Community 内でディスカッションに参加してください。
