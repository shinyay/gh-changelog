# GitHub Projects で階層ビューが利用可能に

## Overview
GitHub Projects の階層ビューがパブリックプレビューで利用可能になりました。プロジェクトのテーブルビューで完全な issue 階層を直接表示できるようになり、コンテキストを失ったりビューを切り替えたりすることなく、複雑な作業の内訳を明確に可視化できます。

## Detailed Explanation
### Overview
- GitHub Projects の階層ビューがパブリックプレビューで利用可能になりました。プロジェクトのテーブルビューで完全な issue 階層を直接表示できるようになり、コンテキストを失ったりビューを切り替えたりすることなく、複雑な作業の内訳を明確に可視化できます。

### 🔗 What is hierarchy view?
- 階層ビューにより、GitHub Projects で直接サブ issue の完全なネストを確認できます。
- 階層ビューを有効にすると、次のことができます:
- 最大 8 レベルの深さまでサブ issue を展開および折りたたみます。
- 階層を維持しながらグループ化、スライス、ソート、フィルタリングを行います。
- まだプロジェクトにない既存のサブ issue を表示してすばやく追加します。

### Getting started
- 階層ビューを有効にするには、View メニューを開いて Show hierarchy をオンにします。

### Coming soon
- 階層ビューへの投資を続けています。次に予定されているのは:
- プロジェクトから直接インラインでサブ issue を作成して追加する機能。
- ドラッグ&ドロップによるサブ issue の並べ替えと親の変更機能。
- issue のサブ issue リストとプロジェクトのサブ issue リストの間の同期された順序。

### Tell us what you think
- あなたのフィードバックをお待ちしています。GitHub Community で会話に参加するか、プロジェクトの … メニューから Give feedback を選択して考えを共有してください。

### Performance enhancements to issues
- issue をより高速にすることを続けています。今週、最近表示された issue を瞬時に感じさせることに焦点を当てた最初のパフォーマンス向上を出荷しました。「瞬時」のベンチマークとして 200ms 未満のロード時間を使用すると、このアップデートにより、瞬時にロードされる issue の割合が 2% から 12% に増加しました。
- これは、今後数か月にわたって計画されている、issue の操作を一貫して高速にするためのいくつかのパフォーマンス向上の最初のものです。

### Other improvements
- プロジェクトに新しい View メニューを追加し、ビュー表示オプションを見つけやすくしました。
- プロジェクトビューを作成する新しい REST API エンドポイントがあります。
- マージキュー内のプルリクエストがプロジェクトでマージキューの状態アイコンを表示するようになりました。
- issue の「Development」セクションからプルリクエスト URL を貼り付けて、アイテムをすばやくリンクできるようになりました。

## Key Changes
- プロジェクトに新しい View メニューを追加し、ビュー表示オプションを見つけやすくしました。
- プロジェクトビューを作成する新しい REST API エンドポイントがあります。

- issue の「Development」セクションからプルリクエスト URL を貼り付けて、アイテムをすばやくリンクできるようになりました。

- GitHub Projects の階層ビューがパブリックプレビューで利用可能になりました。
