# 特定のリポジトリに対してシークレットスキャンの拡張メタデータが自動的に有効化される予定

## Overview
GitHub シークレットスキャンは、2026年2月18日からセキュリティ設定での拡張メタデータチェックのサポートを追加しています。

## Detailed Explanation
### Overview
- GitHub シークレットスキャンは、2026年2月18日からセキュリティ設定での拡張メタデータチェックのサポートを追加しています。
- 拡張メタデータチェックは、有効性チェックの下にネストされた機能です。セキュリティ設定を介してリポジトリの有効性チェックをすでに有効にしている場合、拡張メタデータチェックが自動的に有効になります。

### What are extended metadata checks?
- 拡張メタデータチェックにより、シークレットスキャンアラートは、シークレットプロバイダーから情報が利用可能な場合、シークレットの所有者、シークレットの作成日と有効期限、プロジェクトまたは組織のコンテキストに関する詳細を表示するようになりました。この機能は、シークレットに関する追加のコンテキストに有効性チェックを拡張します。
- たとえば、情報が利用可能な漏洩した OpenAI キーは、組織に関する情報に加えて、シークレット所有者の名前、メール、識別子を表示します。
- これらの新しいメタデータキーは、既存の有効性チェックを拡張して、トリアージと修復のためのより実用的なコンテキストを提供し、開発チームとセキュリティチームが露出をより迅速に評価し、修復の優先順位を付けることができるようにします。
- 注: メタデータの可用性は、シークレットプロバイダー、トークンの種類、場合によってはシークレット自体にも依存します。GitHub は利用可能なすべてのメタデータを表示するよう最善を尽くしますが、すべてのキーが常に存在するとは限りません。

### What's changing?
- 拡張メタデータチェックは現在、有効性チェックが有効になっているシークレットスキャンを使用している Enterprise Cloud のお客様が利用できます。この日から、ユーザーはセキュリティ設定を使用して組織レベルとエンタープライズレベルで機能を有効または無効にできるようになります。この時点で、セキュリティ設定で有効性チェックが有効になっているリポジトリには、メタデータチェックが自動的に有効になります。
