# Actions キャッシュエントリのレート制限

## Overview
GitHub Actions キャッシュに、リポジトリあたり 200 アップロード/分のレート制限が設けられました。この制限は新しいキャッシュエントリのアップロードにのみ影響します。ジョブ実行でダウンロードされるキャッシュエントリには影響しません。

## Detailed Explanation
### Overview
- GitHub Actions キャッシュに、リポジトリあたり 200 アップロード/分のレート制限が設けられました。この制限は新しいキャッシュエントリのアップロードにのみ影響します。ジョブ実行でダウンロードされるキャッシュエントリには影響しません。
- この制限は、非常に短期間に大量のキャッシュエントリをアップロードするリポジトリに対応して実装されました。これにより、キャッシュのスラッシングが増加し、システムの安定性に影響を与えます。これらの課題を軽減する動作を使用するために、一部のマーケットプレイスアクションの作成者と協力しています。
- レート制限の影響を受けるユーザーは、レート制限がリセットされるまで、一部の新しいキャッシュエントリのアップロードが拒否されることが予想されます。

## Impact / Who’s Affected
- この制限は新しいキャッシュエントリのアップロードにのみ影響します。ジョブ実行でダウンロードされるキャッシュエントリには影響しません。
