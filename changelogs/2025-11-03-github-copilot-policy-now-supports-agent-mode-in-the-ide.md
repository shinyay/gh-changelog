# GitHub Copilot policy now supports agent mode in the IDE

## Overview
GitHub admins can now manage access to Copilot agent mode for IDE via the Copilot policies page on github.com.

## Detailed Explanation
### Overview
- GitHub admins can now manage access to Copilot agent mode for IDE via the Copilot policies page on github.com.

### What’s changed
- To provide better governance, Copilot policies now allow for enabling and disabling access to Copilot agent mode in the IDE for enterprises and organizations.
- Prior to this update, Copilot agent mode in the IDE was enabled for all users. To prevent disruption, the new policy has shipped with a default setting of Enabled . To modify this setting, please visit the AI Controls tab for your enterprise or Copilot policies tab for your organization.

- For more information on Copilot policies please visit our documentation page for Copilot policies .

## [What’s changed](#whats-changed)


To provide better governance, Copilot policies now allow for enabling and disabling access to Copilot agent mode in the IDE for enterprises and organizations.


Prior to this update, Copilot agent mode in the IDE was enabled for all users. To prevent disruption, the new policy has shipped with a default setting of `Enabled`. To modify this setting, please visit the **AI Controls** tab for your enterprise or **Copilot policies** tab for your organization.
