# Billing date standardized to the first of the month for self-serve credit card metered Enterprise customers now generally available

## Overview
Starting December 1, 2025 , all usage-based GitHub products paid by credit card on self-serve metered GitHub Enterprise Cloud accounts will be billed on the first of each month. Your billing period will remain the calendar month. No action is needed—this update will happen automatically.

## Detailed Explanation
### Overview
- Starting December 1, 2025 , all usage-based GitHub products paid by credit card on self-serve metered GitHub Enterprise Cloud accounts will be billed on the first of each month. Your billing period will remain the calendar month. No action is needed—this update will happen automatically.
- This applies to metered GitHub products, including:
- GitHub Actions (minutes used, storage)
- GitHub Codespaces (compute hours, storage)
- GitHub Packages (storage, data transfer)
- Advanced Security (active committer seats)
- Copilot Business/Enterprise (metered seat overages)
- Any additional storage or bandwidth charges
- To help keep you informed, we are placing a banner on the Billing Overview page about this upcoming change.
- Important: Any payment processed in November, regardless of the payment date, was related to usage in October. The December 1 payment will include all November usage. It is not a duplicate payment.
- Why this change? – Get a more predictable, consistent billing schedule. – Improve billing clarity with a standard payment date.
- If you wish to opt out of this change, you’ll need to switch to invoiced billing. Contact Sales for assistance.

## Caveats / Limitations
- Important: Any payment processed in November, regardless of the payment date, was related to usage in October.

This applies to metered GitHub products, including:


* GitHub Actions (minutes used, storage)
* GitHub Codespaces (compute hours, storage)
* GitHub Packages (storage, data transfer)
* Advanced Security (active committer seats)
* Copilot Business/Enterprise (metered seat overages)
* Any additional storage or bandwidth charges


To help keep you informed, we are placing a banner on the Billing Overview page about this upcoming change.


**Important:** Any payment processed in November, regardless of the payment date, was related to usage in October. The December 1 payment will include all November usage. It is not a duplicate payment.


**Why this change?**  

– Get a more predictable, consistent billing schedule.  

– Improve billing clarity with a standard payment date.


If you wish to opt out of this change, you’ll need to switch to invoiced billing. [Contact Sales](https://github.com/contact/sales) for assistance.
