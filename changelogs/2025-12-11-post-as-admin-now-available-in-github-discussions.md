# Post as Admin now available in GitHub Discussions

## Overview
Repository administrators can now post and comment in GitHub Discussions using an “Admin” badge to distinguish official responses from personal contributions.

## Detailed Explanation
### Overview
- Repository administrators can now post and comment in GitHub Discussions using an “Admin” badge to distinguish official responses from personal contributions.
- The option to post as Admin appears when creating discussions or adding comments. This allows administrators to choose when their response represents official repository guidance. This helps community members easily identify authoritative answers while maintainers retain the flexibility to personally participate in conversations. The feature eliminates the need for separate admin accounts — a common workaround that introduces security risks and complicates access management.

### How to post as Admin
- To post using the administrator alias:
- Have administrator permissions on the repository.
- Enable GitHub Discussions for the repository.
- Select Admin when creating or replying to discussions.
- Learn more about managing discussions for your community .
- This feature is now available on github.com for all repositories with discussions enabled.

## Impact / Who’s Affected
- This feature is now available on github.com for all repositories with discussions enabled.
