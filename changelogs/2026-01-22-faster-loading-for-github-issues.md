# Faster loading for GitHub Issues

## Overview
GitHub Issues are now significantly faster with 35% of issue views occurring in under 200ms, up from 2% at the start of the year.

## Detailed Explanation
### Overview
- GitHub Issues are now significantly faster with 35% of issue views occurring in under 200ms, up from 2% at the start of the year.
- This is the first in a series of performance improvements we’re shipping to make GitHub Issues, and GitHub more broadly, feel instant.

### Availability
- These performance improvements are live today on github.com. In order to benefit from them, you must be signed in . No configuration or opt-in is required.
- You can experience these improvements through:
- The repository index
- Your “Issues” dashboard
- A GitHub Project
- We’re working on bringing these improvements to additional pathways.

- Join the GitHub Community to share your feedback.
